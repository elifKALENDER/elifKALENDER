### 🏗️ Architecture × AI — Where Design Meets Code

I'm an architect who builds **custom AI pipelines** for the architecture and design industry.  
My work sits at the intersection of architectural domain knowledge and machine learning — turning early-stage sketches, massing models, and reference images into **photorealistic, design-faithful renders**.

I don't just use AI tools. I build them.

---

### 🔧 Signature Workflows

| Project | Description | Status |
|---|---|---|
| [**ImageToRender**](https://github.com/elifKALENDER/ImageToRender) | Generate material, lighting & seasonal variants from a single reference image | ✅ Public |
| **SketchToRender** | Hand-drawn concept sketches → photorealistic visuals, preserving design intent | 🔒 Private |
| **MassToRender** | 3D massing models → detailed architectural context with accurate scale and volume | 🔒 Private |

> These aren't generic AI outputs. Each workflow is fine-tuned to respect architectural constraints.

---

### 🚀 Currently Building

**Architectural AI Suite — Unity Application**  
Bringing SketchToRender, ImageToRender, and MassToRender into a single Unity-based desktop application. Python workflows connected to a clean UI — so architects can run AI renders without touching the terminal.

`Python` `Unity` `C#` `Stable Diffusion` `ComfyUI` → desktop app pipeline
